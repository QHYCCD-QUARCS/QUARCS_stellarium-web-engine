# 导星画框与星点数据标注流程

最后更新：2026-06-28

## 1. 文档目的

本文说明“导星预览中的候选星画框、最终候选框、锁定框，以及框旁边显示的星点数据文字”这一整条链路，覆盖：

- 后端如何计算候选星与最终选星
- 后端如何把位置和文字通过 WebSocket 发给前端
- 前端如何把图像坐标换算成屏幕坐标
- 前端如何最终绘制出框和文字

当前以**实际保留并生效的 DOM overlay 方案**为准。

说明：

- 之前前端曾同时存在 canvas 直画和 DOM overlay 两套实现
- 现已确认保留 `gui.vue` 的 DOM overlay 路径
- `App.vue` 中 canvas 直画调试标注函数已删除，不再参与最终绘制

---

## 2. 当前实际生效的整体流程

当前流程可以概括为：

1. `GuidingStarDetector::selectGuideStar()` 在后端对当前导星帧做候选星检测、筛选与最终选星
2. `GuiderCore` 把候选星列表、最终候选星标签、已选中的星点坐标整理成调试叠加数据
3. `MainWindow` 监听 `GuiderCore` 信号，并通过 WebSocket 发给前端
4. 前端 `App.vue` 接收这些消息
5. `App.vue` 把“图像像素坐标”映射为“屏幕绝对定位坐标”
6. `App.vue` 再通过事件总线把换算后的框位置、框尺寸、标签文字广播给 `gui.vue`
7. `gui.vue` 使用绝对定位 `div` 把框和标签渲染在导星画布上方

## 3. 后端计算阶段

### 3.1 星点检测与最终选星

核心入口：

- Qt 后端：[GuidingStarDetector.cpp](/home/q/workspace_origin/QUARCS_QT-SeverProgram/src/guiding/GuidingStarDetector.cpp:953)

关键函数：

- `GuidingStarDetector::selectGuideStar(const cv::Mat& image16, ...)`

该函数负责：

- 输入当前导星帧 `image16`
- 先做基础候选星检测
- 再做测量与筛选
- 最终选出一颗 guide star

主要阶段包括：

1. 基础候选星检测
2. 候选星测量
3. SNR / HFD / 边缘 / 质心等过滤
4. 最终选中一颗最佳导星星点

后端日志里也能看到这一阶段的 trace，例如：

- `selectGuideStar START`
- `detectPHD2StylePeaks`
- `measureStarCandidate`
- `finalSelect`
- `selectGuideStar SUCCESS`

### 3.2 GuiderCore 组装调试叠加数据

核心位置：

- [GuiderCore.cpp](/home/q/workspace_origin/QUARCS_QT-SeverProgram/src/guiding/GuiderCore.cpp:1204)

关键逻辑：

- `publishDynamicDebugOverlay`

这里会重新拿到一批导星候选数据，并整理成几组前端可视化数据：

- `dedupCandidates`
  说明：去重后的候选星
- `snrCandidates`
  说明：通过 SNR 阶段的候选星
- `candidates`
  说明：最终候选集合
- `candLabels`
  说明：每个最终候选框旁边要显示的文字

当前候选框旁的文字由后端直接生成，格式在这里写死：

- [GuiderCore.cpp](/home/q/workspace_origin/QUARCS_QT-SeverProgram/src/guiding/GuiderCore.cpp:1249)

具体格式：

- `HFD=%1 SNR=%2`

也就是说，最终候选框旁显示的这段文字不是前端现算的，而是后端先拼好，再发给前端。

### 3.3 锁定星点信息

在真正锁定导星星点后，`GuiderCore` 会发出锁定相关信号，`MainWindow` 会进一步包装成前端消息。

关键位置：

- [mainwindow.cpp](/home/q/workspace_origin/QUARCS_QT-SeverProgram/src/mainwindow.cpp:1378)

这里会发送两类与“当前锁定星点”相关的消息：

1. 搜索框位置消息
   - `PHD2StarBoxPosition:imageW:imageH:x:y:halfSize`
2. 锁定星点信息消息
   - `GuiderStarSelected:x=...:y=...:snr=...:hfd=...`

其中：

- `PHD2StarBoxPosition` 用来画当前锁定框
- `GuiderStarSelected` 用来显示“导星星点: (x, y) SNR=... HFD=...”这段文字

### 5.2 图像坐标到屏幕坐标的换算

导星画布的屏幕尺寸获取函数：

- [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10498)

函数：

- `getGuiderCanvasDisplayRect()`

它负责拿到导星 canvas 当前在页面上的：

- `left`
- `top`
- `width`
- `height`

图像坐标映射函数：

- [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10542)

函数：

- `mapGuiderImagePointToScreen(PHD2ImageSize_X, PHD2ImageSize_Y, pointX, pointY)`

换算公式本质上是：

- `screenX = rect.left + pointX * rect.width / imageW`
- `screenY = rect.top  + pointY * rect.height / imageH`

因此：

- 后端给的是导星图像中的像素点
- 前端这里把它映射成浏览器窗口中的绝对定位点

### 5.3 各种框尺寸的换算

`App.vue` 里不同框会用不同的固定逻辑尺寸，再按画布显示比例放大/缩小：

- 搜索框：`resolvedHalfSizePx * 2`
  - [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10535)
- dedup 候选框：30
  - [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10606)
- snr 候选框：40
  - [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10617)
- final 候选框：50
  - [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10638)
- selected 候选框：60
  - [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10650)

然后统一按：

- `displayWidth = logicalWidth * rect.width / imageW`
- `displayHeight = logicalHeight * rect.height / imageH`

换算成屏幕尺寸。

### 5.4 App.vue 把换算结果广播给 gui.vue

换算完成后，`App.vue` 并不自己画，而是把结果继续通过事件总线发给 `gui.vue`。

例如最终候选框：

- [App.vue](/home/q/workspace_origin/QUARCS_stellarium-web-engine/apps/web-frontend/src/App.vue:10638)

会触发：

- `GuiderDebugFinalCandidatePosition(StarStartX, StarStartY, StarWidth, StarHeight, label)`

搜索框则触发：

- `PHD2BoxPosition`

锁定星原始数据会触发：

- `GuiderLockStar`
- `GuiderStarSelected`

## 7. 当前实际生效的消息与职责分工

### 7.1 后端负责

- 检测候选星
- 筛选最终候选星
- 选出锁定星
- 生成最终候选框旁标签字符串 `HFD=... SNR=...`
- 通过 WebSocket 发出图像坐标和标签文本

### 7.2 前端 App.vue 负责

- 接收 WebSocket 消息
- 将导星图像坐标映射为屏幕绝对定位坐标
- 将换算好的位置和尺寸通过事件总线广播给 `gui.vue`

### 7.3 前端 gui.vue 负责

- 用绝对定位 DOM 画框
- 显示最终候选框旁边的标签文字
- 显示锁定框旁的“导星星点”信息

## 9. 现状结论

当前“画框 + 框旁星点数据文字”的生效路径是：

- 后端 `GuidingStarDetector` / `GuiderCore` 负责算星和产出标签
- 后端 `MainWindow` 负责把候选框位置和标签发给前端
- 前端 `App.vue` 负责把图像坐标映射到屏幕坐标
- 前端 `gui.vue` 负责真正把框和文字画出来

当前已经确认：

- 保留方案：`gui.vue` DOM overlay
- 已移除方案：`App.vue` canvas 直画调试叠加

如果后续需要继续改这条链路，推荐优先从以下几个点进入：

- 后端候选星/标签生成：`GuidingStarDetector.cpp`、`GuiderCore.cpp`
- 后端消息发送：`mainwindow.cpp`
- 前端坐标换算：`App.vue`
- 前端最终样式和文案：`gui.vue`
